GitHub Pages vs Surge for Launching Static Sites

Comparing the two:

Steps for GitHub Pages:

  1. First off open up your browser
  2. Go to github.com
  3. Create an account
  4. Once you have an account, log in on github.com
  5. Then you will need to install the latest version of Git
  6. After installing, open up your Terminal
  7. Run the command: git config --global user.name "YOUR NAME"
  8. Run the command: git config --global user.email "YOUR EMAIL ADDRESS"
  9. You will then need to authenticate with Github from Git
  10. To authenticate, you need to generate SSH keys for each computer you will be using
  11. First check for existing SSH keys--Run the command: ls -al ~/.ssh
  12. Generate a new SSH key--Run the command: ssh-keygen -t rsa -C "your_email@example.com"
  13. Enter a file to save the key (default settings are recommended) [press enter]
  14. You'll then be asked to enter a passphrase
  15. Re-enter passphrase
  16. You'll then be given the ID to your SSH key
  17. Ensure SSH agent is enabled--Run the command: $ eval "$(ssh-agent -s)"
  18. Add your generated SSH key to the ssh-agent--Run the command: ssh-add ~/.ssh/id_rsa
  19. Copy the SSH key to your clipboard for adding to your account--Run the command: pbcopy < ~/.ssh/id_rsa.pub
  20. Open up your browser
  21. Go to github.com
  22. Login to your account
  23. In the top right corner of any page click the user settings token/icon
  24. In the user settings sidebar click SSH keys
  25. Click Add SSH key
  26. In the Title field add a descriptive label for the new key. e.g. "Personal Macbook Air"
  27. Paste your key into the "Key" field
  28. Click Add Key
  29. Confirm the action by entering your GitHub password
  30. Open your terminal to test the connection
  31. Run the command: ssh -T git@github.com
  32. You may be prompted with a warning--verify the RSA key fingerprint in the warning message and type yes
  33. If the username in the following greeting message matches your username then you've successfully set up your SSH key
  34. Open up your Terminal
  35. cd path/to/myproject
  36. git init
  37. git add .
  38. git commit -m "my first commit"
  39. git remote add origin git@github.com:me/my-project.git
  40. git checkout -b gh-pages
  41. git push origin gh-pages

Steps for Surge:

  1. Open web browser
  2. Download and Install node.js
  3. Open up your Terminal
  4. Install surge by running: npm install --global surge
  5. launch project by running: surge path/to/myproject myproject.surge.sh

    6. Conclusion:

    I recently joined the team that builds Surge as a practicum student and was tasked with comparing the workflow of launching a static web application through Surge vs GitHub Pages. Surge has separated itself by simplifying the process as a dedicated CDN through the terminal. I found that after reading the docs the implementation was relatively straight forward after downloading and installing node.js. The commands require a file path to your project and a domain name for an initial launch. Whether that is a subdomain of surge.sh or linking to your custom domain through a CNAME file. GitHub Pages requires a lot more work for a new user to setup the environment between Git and GitHub including authenticating through an SSH key and setting up GitHub Pages branches. This would be a deterrent for some novice web developers who are using a CDN to launch their first project. Overall, having a simple terminal based launch in 3 steps is what gives Surge the edge for launching static sites for free.
